핵심 요구사항 분석

  • 115문제 진단 테스트 결과 데이터 취합·가공 및 외부 시각화 서비스 전송 API/JSON 규격 개발
  • 외부 BI/로우코드 솔루션 내 도표 및 상세 분석 화면 구축
  • '뉴런 문법시험' 학생별 일일 통과 여부 통합 조회 화면 + 관리 기능 신규 개발
  • 요구사항 정의서, 연동 규격서 등 기획 산출물 제공
  • 운영 중인 레거시 Java/Thymeleaf 시스템 코드 분석 및 영향 범위 최소화 개발
  • 외부 시각화 서비스 연동 가이드 및 화면 세팅 내역서 작성

제안 포인트

  1. 레거시 분석 우선 접근: 운영 중인 LMS 영향을 최소화하기 위해 첫 주는 기존 DB 스키마와 데이터 흐름을 정밀 분석합니다. 기존 테이블 구조를 변경하지 않고 신규 집계 테이블/뷰를 추가하는 방식으로 비파괴적 고도화를 진행합니다.
  2. BI 솔루션 독립적인 표준 연동 규격: 외부 시각화 서비스를 Metabase, Looker Studio, Apache Superset 등 어느 것을 선택하더라도 재사용 가능한 표준 JSON 스키마와 REST API 인터페이스로 설계합니다. 향후 BI 솔루션 교체 시에도 LMS 측 수정 없이 대응 가능합니다.
  3. 어드민 UX 일관성 유지: 신규 통합 조회·관리 화면을 기존 Thymeleaf 템플릿 패턴과 동일하게 구성하여 관리자가 별도 학습 비용 없이 즉시 사용할 수 있도록 합니다.
  4. 1순위 산출물 완성도에 맞춘 마일스톤 운영: 매주 검수 가능한 산출물 단위로 4개 마일스톤(기획 문서, 외부 연동 데모, 어드민+BI 세팅, 최종 인도)을 분할하여 진행 상황을 실시간 가시화합니다.
  5. 장기 파트너십을 고려한 문서화: 우대 사항 중 '장기적 파트너십'을 반영하여 인수인계 가능한 수준의 연동 규격서·운영 가이드를 제공합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

현재 운영 중인 영어 문법 학습 LMS는 학생들에게 115문제 진단 테스트 및 매일 진행되는 '뉴런 문법시험'을 제공하고 있습니다. 다음 두 가지 운영상 어려움이 존재합니다:

  • 진단 결과 분석 한계: 텍스트·표 위주 결과로는 학생별 약점 영역을 직관적으로 파악하기 어려움
  • 일일 평가 관리 부담: 학생별 매일 통과 여부를 일일이 확인해야 하므로 관리자 운영 시간 과다 소요

본 프로젝트는 외부 시각화 솔루션 연동과 관리자 통합 조회 화면 신규 개발로 두 문제를 동시에 해결합니다.

3. 기능 요구사항

3.1 진단 테스트 결과 외부 연동

  • [필수] 115문제 응답 데이터 집계 (학생별·문항별·영역별)
  • [필수] 외부 시각화 서비스 전송용 REST API 제공 (JSON Schema 명세 포함)
  • [필수] API 인증 (토큰 기반, 만료/갱신)
  • [필수] 데이터 가공 정책 (백분율, 영역별 정답률, 약점 영역 자동 분류)
  • [선택] 증분 전송 (전체 재전송 대신 변경분만 push)

3.2 외부 시각화 솔루션 화면 구축

  • [필수] BI 툴 데이터소스 연결 (Metabase, Looker Studio, Apache Superset 중 클라이언트 선택)
  • [필수] 학생별 진단 결과 도표 (영역별 정답률, 약점 영역 히트맵, 전체 분포 비교)
  • [필수] 상세 분석 화면 (문항별 응답 분포, 시계열 추이)
  • [필수] 화면 세팅 내역서 작성 (재구축·재설정 가능 수준)

3.3 관리자 대시보드 고도화

  • [필수] '뉴런 문법시험' 학생별 일일 통과 통합 조회 화면 (날짜·반·학생 필터)
  • [필수] 통과/미통과 일괄·개별 처리 기능
  • [필수] CSV 다운로드 (일일·기간별 리포트)
  • [선택] 미통과 학생 자동 알림 트리거 (이메일/SMS 훅)

4. 비기능 요구사항

항목요구사항
성능외부 BI 전송 API 응답 200ms 이내 (학생 500명 기준), 통합 조회 화면 1초 이내 렌더
보안HTTPS 강제, API 토큰 만료/갱신, SQL Injection 방어, 학생 개인정보 마스킹
호환성기존 Java/Thymeleaf 코드와 100% 호환, 신규 의존성 최소화
확장성BI 솔루션 교체 가능한 표준 JSON 스키마, 학생 수 1만 명까지 무리 없는 처리
가용성기존 LMS 운영 영향 없음 (무중단 배포)

5. 기술 제약사항

  • 기술 스택 고정: 기존 Java + Thymeleaf, MySQL 유지 (프레임워크 교체 불가)
  • 레거시 DB 스키마 보존: 기존 테이블 구조 변경 최소화, 신규 집계 테이블/뷰 추가 방식 우선
  • 운영 무중단: 운영 중인 LMS에 영향 없이 점진 배포
  • 외부 BI 도구 라이선스: 무료/오픈소스(Metabase OSS, Apache Superset, Looker Studio Free) 중 클라이언트 선택

6. 범위 외 사항 (Out of Scope)

  • 학생 앱/사용자 화면 신규 개발
  • 진단 테스트 문항 자체 변경/추가
  • LMS 인증/회원 시스템 리뉴얼
  • 모바일 전용 반응형 (PC Web 한정)
  • BI 솔루션 자체 라이선스 비용 부담
